# Security And Private Data

Report issues by opening a GitHub issue with reproduction steps and public-safe artifacts only.

Do not attach:

- Pokemon ROMs;
- private save files;
- private `.red.json` files;
- screenshots containing private data;
- emulator state directories;
- personal filesystem paths;
- secrets or credentials.

If a bug requires a private save to reproduce, describe the semantic shape of the save and the failing CLI command. Share private artifacts only through a mutually agreed private channel.
